HTML Semântico

HTML semântico descreve o significado do conteúdo, não sua aparência. Ele melhora acessibilidade, SEO, manutenção e leitura por máquinas.

O que é semântica?

Semântica é o uso de elementos HTML que expressam o papel do conteúdo.

  • Para leitores de tela
  • Para mecanismos de busca
  • Para desenvolvedores

Regra: se a tag tem significado, use-a.

<header> — Cabeçalho

Representa o cabeçalho de uma página ou seção. Pode conter títulos, logos, menus e introduções. 

<header>
  <h1>Site</h1>
</header>

Observações:

  • Pode existir mais de um <header>
  • Não é sinônimo de topo visual

<nav> — Navegação

Agrupa links de navegação principais ou secundários. 

<nav>
  <a href="#">Home</a>
  <a href="#">Contato</a>
</nav>

Use apenas para blocos de navegação.

<main> — Conteúdo principal

Representa o conteúdo central e único da página. 

<main>
  <article>Conteúdo</article>
</main>

Regras:

  • Apenas um <main> por página
  • Não deve estar dentro de header, footer ou aside

<section> — Seção temática

Agrupa conteúdo relacionado por tema. Normalmente possui um título. 

<section>
  <h2>Notícias</h2>
</section>

Se não há título, talvez não seja uma section.

<article> — Conteúdo independente

Representa conteúdo que pode existir sozinho: post, notícia, comentário, card. 

<article>
  <h2>Post</h2>
  <p>Texto</p>
</article>

Dica: se pode ser reutilizado fora da página, é um article.

<aside> — Conteúdo complementar

Conteúdo relacionado, mas não essencial. 

<aside>
  Conteúdo lateral
</aside>

Exemplos: barra lateral, widgets, links relacionados.

<footer> — Rodapé

Representa o rodapé de uma página ou seção. 

<footer>
  <p>© 2026</p>
</footer>

Pode conter: autor, copyright, links legais.

<address> — Informações de contato

Usado para informações do autor ou responsável pelo conteúdo. 

<address>
  Contato: email@exemplo.com
</address>

Não é para endereços genéricos aleatórios.

<figure> e <figcaption>

Agrupam mídia com legenda associada. 

<figure>
  <img src="img.png">
  <figcaption>Legenda</figcaption>
</figure>

Legenda faz parte do conteúdo, não é decorativa.

<time> — Datas e horários

Representa datas legíveis por máquinas. 

<time datetime="2026-02-10">10 de fevereiro</time>

Usado para eventos, posts e históricos.

<details> e <summary>

Criam conteúdo expansível nativo. 

<details>
  <summary>Mais informações</summary>
  Texto oculto
</details>

Interativo sem JavaScript.

Erros comuns

  • Usar <div> para tudo
  • Usar <section> sem título
  • Confundir article com section
  • Ignorar semântica por estética

Resumo

  • HTML semântico descreve significado
  • CSS cuida da aparência
  • Boa semântica melhora tudo
