Listas em HTML

Listas são usadas para representar conjuntos de itens relacionados. O HTML possui três tipos principais de listas, cada uma com significado semântico específico.

<ul> — Lista não ordenada

Representa uma lista onde a ordem dos itens não importa. Visualmente, costuma ser exibida com marcadores (bolinhas). 

<ul>
  <li>Item A</li>
  <li>Item B</li>
  <li>Item C</li>
</ul>

Uso comum:

  • Menus
  • Listas de opções
  • Agrupamento de informações

Erro comum: usar <br> no lugar de listas.

<ol> — Lista ordenada

Representa uma lista onde a ordem é relevante. Por padrão, é numerada. 

<ol>
  <li>Primeiro passo</li>
  <li>Segundo passo</li>
  <li>Terceiro passo</li>
</ol>

Atributos importantes

type

Define o tipo de numeração. 

<ol type="1">  
<ol type="A">  
<ol type="a">  
<ol type="I">  
<ol type="i">

start

Define o número inicial da lista. 

<ol start="5">

reversed

Inverte a ordem da contagem. 

<ol reversed>

<li> — Item de lista

Define cada item dentro de <ul> ou <ol>. 

<li>Item</li>

Um <li> pode conter:

  • Texto
  • Outras listas
  • Links
  • Elementos HTML em geral

Listas aninhadas

É possível criar listas dentro de outras listas, mantendo uma hierarquia clara. 

<ul>
  <li>Front-end
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
</ul>

Boa prática: respeitar a hierarquia e evitar excessos.

<dl> — Lista de definição

Usada para relacionar termos e suas descrições. Muito comum em dicionários e glossários. 

<dl>
  <dt>HTML</dt>
  <dd>Linguagem de marcação</dd>
</dl>

Tags envolvidas

  • <dl> — lista de definição
  • <dt> — termo
  • <dd> — descrição

Um mesmo termo pode ter várias descrições.

Acessibilidade e semântica

Listas bem estruturadas ajudam:

  • Leitores de tela
  • Mecanismos de busca
  • Manutenção do código

Dica: nunca use listas apenas por aparência visual.
