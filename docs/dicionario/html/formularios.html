Formulários no HTML

Formulários permitem a entrada de dados pelo usuário. São usados para login, cadastro, busca, envio de mensagens e interação com sistemas web.

<form> — Contêiner do formulário

A tag <form> envolve todos os campos do formulário e define como os dados serão enviados. 

<form action="processar.php" method="post">
</form>

Atributos principais

  • action — destino dos dados
  • method — método HTTP (get ou post)
  • autocomplete — ativa ou desativa preenchimento automático

<label> — Rótulo de campo

Associa um texto explicativo a um campo de formulário. É essencial para acessibilidade. 

<label for="email">E-mail</label>
<input id="email">

Boa prática: sempre usar label com for.

<input> — Campo de entrada

Elemento mais versátil dos formulários. Seu comportamento depende do atributo type. 

<input type="text">

Tipos comuns

  • text — texto simples
  • email — e-mail
  • password — senha
  • number — números
  • date — data
  • checkbox — múltipla escolha
  • radio — escolha única
  • file — envio de arquivos
  • hidden — campo invisível

Atributos importantes do <input>

<input
  type="text"
  name="usuario"
  placeholder="Digite seu nome"
  required
  minlength="3"
  maxlength="20"
>
  • name — nome do campo (obrigatório para envio)
  • placeholder — dica visual
  • required — campo obrigatório
  • readonly — somente leitura
  • disabled — desativado

<textarea> — Texto longo

Usado para mensagens, comentários ou descrições. 

<textarea rows="4" cols="40"></textarea>

Boa prática: controlar tamanho com CSS, não com cols.

<select> e <option> — Lista suspensa

Permite selecionar uma opção entre várias. 

<select name="cidade">
  <option value="rs">Rio Grande do Sul</option>
  <option value="sc">Santa Catarina</option>
</select>

<optgroup>

<select>
  <optgroup label="Sul">
    <option>RS</option>
    <option>SC</option>
  </optgroup>
</select>

<button> — Botões

Representa ações dentro do formulário. 

<button type="submit">Enviar</button>
<button type="reset">Limpar</button>
<button type="button">Ação</button>

Prefira <button> a <input type="submit">.

<fieldset> e <legend>

Agrupam campos relacionados. 

<fieldset>
  <legend>Dados pessoais</legend>
  <input type="text">
</fieldset>

Validação nativa

O HTML oferece validação automática sem JavaScript. 

<input type="email" required>
<input type="number" min="1" max="10">

O navegador bloqueia o envio se os dados forem inválidos.

Boas práticas em formulários

  • Usar <label> corretamente
  • Agrupar campos com <fieldset>
  • Evitar placeholders como substituto de labels
  • Fornecer mensagens claras de erro
