Interações e Movimento

CSS permite criar experiências dinâmicas através de **transições** (mudanças suaves de estado), **transformações** (alteração de forma e posição) e **animações** (sequências complexas de movimentos).

Transitions — Transições Suaves

Permite que as mudanças de propriedades ocorram de forma gradual ao longo de um tempo determinado, em vez de uma mudança instantânea. 

button {
  background-color: blue;
  transition: background-color 0.3s ease-in-out;
}

button:hover {
  background-color: red;
}

Propriedades da Transição

  • transition-property — Qual propriedade será animada (ex: color, width, all).
  • transition-duration — Duração (ex: 0.5s, 200ms).
  • transition-timing-function — Curva de aceleração (linear, ease, ease-in, ease-out, ease-in-out).
  • transition-delay — Tempo de espera antes de começar.

Shorthand: transition: property duration timing delay;

transform — Transformações 2D e 3D

Modifica o espaço de coordenadas do elemento. **Não afeta o layout** ao redor (não empurra outros elementos). 

.box:hover {
  transform: scale(1.2) rotate(45deg);
}

Funções Comuns

  • translate(x, y) — Move o elemento nos eixos X e Y.
  • scale(x, y) — Altera o tamanho (1 = 100%, 1.5 = 150%).
  • rotate(deg) — Rotaciona em graus (ex: 90deg, -15deg).
  • skew(x, y) — Inclina o elemento.

transform-origin: Define o ponto central da transformação (ex: top left, center).

CSS Animations — Animações Complexas

Diferente das transições, as animações não precisam de um gatilho (como hover) e podem ter vários estágios definidos por @keyframes.

1. Definindo a sequência (@keyframes)

@keyframes flutuar {
  0%   { transform: translateY(0); }
  50%  { transform: translateY(-20px); }
  100% { transform: translateY(0); }
}

2. Aplicando a animação

.elemento {
  animation: flutuar 3s infinite ease-in-out;
}

Propriedades de Animação

  • animation-name — Nome do keyframe.
  • animation-duration — Tempo total de um ciclo.
  • animation-iteration-count — Quantas vezes (número ou infinite).
  • animation-direction — normal, reverse, alternate (vai e volta).
  • animation-fill-mode — forwards (mantém o estado final) ou backwards.
  • animation-play-state — running ou paused.

Estados de Interação (Pseudoclasses)

Definem estilos baseados no comportamento do usuário.

  • :hover — Cursor sobre o elemento.
  • :active — Momento do clique.
  • :focus — Quando o elemento recebe foco (via teclado/Tab ou clique em inputs).
  • :visited — Links já clicados.

Performance e Boas Práticas

Animações podem pesar no processamento. Siga estas diretrizes técnicas:

  • Prefira Transform e Opacity: São propriedades otimizadas pela GPU (placa de vídeo). Animar width ou margin causa "reflow" e pode travar a tela.
  • Will-change: Informe ao navegador que um elemento vai mudar (use com moderação): will-change: transform;.
  • Acessibilidade: Respeite usuários que preferem movimentos reduzidos usando a Media Query @media (prefers-reduced-motion: reduce).

cursor — Estilo do Ponteiro

Define qual ícone o mouse exibirá ao passar sobre o elemento.

.clicavel { cursor: pointer; }
.bloqueado { cursor: not-allowed; }
.carregando { cursor: wait; }
